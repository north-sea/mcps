# Data Model: hermes-db WeChat Publication Ledger

**Workspace**: `hermes-db-wechat-publication-ledger` | **Date**: 2026-06-03

---

## Entities

### WeChat Article (表名: `hermes.wechat_articles`)

**描述**: One durable article ledger record for a WeChat publication attempt or successful publication. It is the stable join point for topic, workflow run, generated artifacts, publication result, future analytics snapshots, and repair workflows.

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| article_id | UUID | PRIMARY KEY | Server-generated stable article id, generated by repository/tool code |
| publication_idempotency_key | TEXT | NOT NULL | Logical publication identity supplied by client or derived by tool |
| account | TEXT | NOT NULL, INDEX | WeChat account alias |
| topic_id | UUID | FK -> `hermes.topics(id)`, NULL, INDEX | Related topic when available |
| run_id | TEXT | FK -> `hermes.wechat_workflow_runs(run_id)`, NOT NULL, INDEX | Workflow run that produced/published this article |
| task_id | TEXT | NULL, INDEX | Optional downstream task id |
| draft_artifact_id | TEXT | FK -> `hermes.workflow_artifacts(artifact_id)`, NULL | Original draft artifact |
| published_artifact_id | TEXT | FK -> `hermes.workflow_artifacts(artifact_id)`, NULL | Final text artifact used for publication |
| publish_artifact_id | TEXT | FK -> `hermes.workflow_artifacts(artifact_id)`, NULL | Publish result artifact |
| status | TEXT | NOT NULL, CHECK | Article ledger status |
| dry_run | BOOLEAN | NOT NULL, DEFAULT false | Whether this article was produced by dry-run |
| title | TEXT | NULL | Article title when known |
| published_url | TEXT | NULL, INDEX | Raw/current published URL |
| canonical_url | TEXT | NULL, INDEX | Caller-provided canonical URL |
| publish_target | TEXT | NULL, INDEX | Target channel/reference, e.g. `wechat-mp`, `youmind` |
| external_reference | TEXT | NULL, INDEX | Current primary external reference when URL is absent |
| metadata | JSONB | NOT NULL, DEFAULT `'{}'::jsonb` | Caller-supplied structured metadata |
| published_at | TIMESTAMPTZ | NULL, INDEX | Publication time when known |
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT now() | Row creation time |
| updated_at | TIMESTAMPTZ | NOT NULL, DEFAULT now() | Row update time |

**状态值**:

```text
drafted
published
published_missing_url
publish_reference_missing
archived
```

**状态语义**:

```text
drafted -> published
drafted -> published_missing_url
drafted -> publish_reference_missing
published_missing_url -> published
publish_reference_missing -> published_missing_url
publish_reference_missing -> published
any active status -> archived
```

The database enforces only the allowed status vocabulary. MCP validators enforce status-specific payload rules.

**约束**:

- `uq_wechat_articles_account_idempotency`: unique `(account, publication_idempotency_key)`.
- `chk_wechat_articles_status`: status must be one of the five allowed values.
- `chk_wechat_articles_reference_for_published`: if `status = 'published'`, at least one of `published_url`, `canonical_url`, or `external_reference` must be present.
- FKs validate supplied `run_id` and artifact ids.

**索引**:

- `idx_wechat_articles_account_created` on `(account, created_at DESC)`
- `idx_wechat_articles_account_status_created` on `(account, status, created_at DESC)`
- `idx_wechat_articles_topic_created` on `(topic_id, created_at DESC)`
- `idx_wechat_articles_run_id` on `(run_id)`
- `idx_wechat_articles_task_id` on `(task_id)`
- `idx_wechat_articles_published_url` on `(published_url)` where `published_url IS NOT NULL`
- `idx_wechat_articles_canonical_url` on `(canonical_url)` where `canonical_url IS NOT NULL`
- `idx_wechat_articles_publish_target_created` on `(publish_target, created_at DESC)`
- `idx_wechat_articles_published_at` on `(published_at DESC)` where `published_at IS NOT NULL`

### WeChat Article External Reference (表名: `hermes.wechat_article_external_refs`)

**描述**: Append-friendly reference table for URLs, platform identifiers, YouMind references, and manual repair references. It preserves history while enabling analytics and repair to bind external data back to a stable `article_id`.

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| ref_id | UUID | PRIMARY KEY | Reference row id, generated by repository/tool code |
| article_id | UUID | FK -> `hermes.wechat_articles(article_id)`, NOT NULL, INDEX | Owning article |
| ref_type | TEXT | NOT NULL, CHECK, INDEX | Reference category |
| ref_value | TEXT | NOT NULL | Reference value |
| ref_source | TEXT | NULL | Source of the ref, e.g. `publisher`, `analytics`, `repair` |
| is_primary | BOOLEAN | NOT NULL, DEFAULT false | Whether this is the preferred active ref for its type |
| metadata | JSONB | NOT NULL, DEFAULT `'{}'::jsonb` | Extra parsed fields or provenance |
| superseded_at | TIMESTAMPTZ | NULL | Marks ref as historical but preserved |
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT now() | Row creation time |
| updated_at | TIMESTAMPTZ | NOT NULL, DEFAULT now() | Row update time |

**ref_type 值**:

```text
published_url
canonical_url
wechat_msg_id
wechat_biz_mid_idx_sn
youmind_ref
publish_target_ref
manual_repair
external_reference
```

**约束**:

- `chk_wechat_article_external_refs_type`: `ref_type` must be one of the allowed values.
- `chk_wechat_article_external_refs_value_nonempty`: `length(trim(ref_value)) > 0`.
- `uq_wechat_article_external_ref_active`: unique `(ref_type, ref_value)` where `superseded_at IS NULL`.
- `uq_wechat_article_external_ref_article_active`: unique `(article_id, ref_type, ref_value)` where `superseded_at IS NULL`.

**索引**:

- `idx_wechat_article_refs_article_created` on `(article_id, created_at DESC)`
- `idx_wechat_article_refs_type_value_active` on `(ref_type, ref_value)` where `superseded_at IS NULL`
- `idx_wechat_article_refs_primary` on `(article_id, ref_type, is_primary)` where `superseded_at IS NULL`

## Relationships

```text
hermes.topics 1:N hermes.wechat_articles
hermes.wechat_workflow_runs 1:N hermes.wechat_articles
hermes.workflow_artifacts 1:N hermes.wechat_articles (draft_artifact_id)
hermes.workflow_artifacts 1:N hermes.wechat_articles (published_artifact_id)
hermes.workflow_artifacts 1:N hermes.wechat_articles (publish_artifact_id)
hermes.wechat_articles 1:N hermes.wechat_article_external_refs
```

Relationship behavior:

- `topic_id` uses `ON DELETE SET NULL`; article history must survive topic cleanup.
- `run_id` uses a strong FK. Workflow run deletion is out of scope; if a future retention feature deletes runs, it must decide article retention explicitly.
- Artifact id columns use strong FKs with nullable fields. If a field is supplied, the referenced artifact must exist. Tool validation decides which artifact refs are required for a specific status.
- External refs use `ON DELETE CASCADE` from article to refs, but article deletion is not exposed in MVP.

## Idempotency Rules

### Primary Rule

`(account, publication_idempotency_key)` identifies one logical article publication fact.

```text
same account + same publication_idempotency_key -> return same article
same active external ref claimed by another article -> conflict
new key + no conflicting refs -> create article
```

### Key Derivation Fallback

Clients should pass `publication_idempotency_key`. If missing, MCP tool derives one in this order:

```text
account + publish_target + canonical_url
account + publish_target + external_reference
account + run_id + publish_artifact_id
account + run_id + published_artifact_id
```

If none of these inputs exist, the tool returns `missing_required_field` for `publication_idempotency_key`.

---

## Status-Specific Validation

Database CHECK constraints stay conservative. MCP tool validation should enforce:

| status | Required / Allowed Semantics |
|---|---|
| `drafted` | `dry_run=true` is allowed; artifact refs may be partial; URL refs are optional |
| `published` | at least one of `published_url`, `canonical_url`, `external_reference`, or structured external refs is required |
| `published_missing_url` | publication succeeded but URL is not available; requires `external_reference` or a publish artifact/metadata reason |
| `publish_reference_missing` | publication result lacks stable URL/reference; must preserve repair metadata |
| `archived` | should not delete refs or artifacts |

## DDL Scripts

```sql
CREATE TABLE IF NOT EXISTS hermes.wechat_articles (
    article_id UUID PRIMARY KEY,
    publication_idempotency_key TEXT NOT NULL,
    account TEXT NOT NULL,
    topic_id UUID REFERENCES hermes.topics(id) ON DELETE SET NULL,
    run_id TEXT NOT NULL REFERENCES hermes.wechat_workflow_runs(run_id),
    task_id TEXT,
    draft_artifact_id TEXT REFERENCES hermes.workflow_artifacts(artifact_id),
    published_artifact_id TEXT REFERENCES hermes.workflow_artifacts(artifact_id),
    publish_artifact_id TEXT REFERENCES hermes.workflow_artifacts(artifact_id),
    status TEXT NOT NULL,
    dry_run BOOLEAN NOT NULL DEFAULT false,
    title TEXT,
    published_url TEXT,
    canonical_url TEXT,
    publish_target TEXT,
    external_reference TEXT,
    metadata JSONB NOT NULL DEFAULT '{}'::jsonb,
    published_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    CONSTRAINT uq_wechat_articles_account_idempotency
        UNIQUE (account, publication_idempotency_key),
    CONSTRAINT chk_wechat_articles_status
        CHECK (status IN (
            'drafted',
            'published',
            'published_missing_url',
            'publish_reference_missing',
            'archived'
        )),
    CONSTRAINT chk_wechat_articles_reference_for_published
        CHECK (
            status <> 'published'
            OR published_url IS NOT NULL
            OR canonical_url IS NOT NULL
            OR external_reference IS NOT NULL
        )
);

CREATE INDEX IF NOT EXISTS idx_wechat_articles_account_created
ON hermes.wechat_articles(account, created_at DESC);

CREATE INDEX IF NOT EXISTS idx_wechat_articles_account_status_created
ON hermes.wechat_articles(account, status, created_at DESC);

CREATE INDEX IF NOT EXISTS idx_wechat_articles_topic_created
ON hermes.wechat_articles(topic_id, created_at DESC);

CREATE INDEX IF NOT EXISTS idx_wechat_articles_run_id
ON hermes.wechat_articles(run_id);

CREATE INDEX IF NOT EXISTS idx_wechat_articles_task_id
ON hermes.wechat_articles(task_id);

CREATE INDEX IF NOT EXISTS idx_wechat_articles_published_url
ON hermes.wechat_articles(published_url)
WHERE published_url IS NOT NULL;

CREATE INDEX IF NOT EXISTS idx_wechat_articles_canonical_url
ON hermes.wechat_articles(canonical_url)
WHERE canonical_url IS NOT NULL;

CREATE INDEX IF NOT EXISTS idx_wechat_articles_publish_target_created
ON hermes.wechat_articles(publish_target, created_at DESC);

CREATE INDEX IF NOT EXISTS idx_wechat_articles_published_at
ON hermes.wechat_articles(published_at DESC)
WHERE published_at IS NOT NULL;

CREATE TABLE IF NOT EXISTS hermes.wechat_article_external_refs (
    ref_id UUID PRIMARY KEY,
    article_id UUID NOT NULL REFERENCES hermes.wechat_articles(article_id) ON DELETE CASCADE,
    ref_type TEXT NOT NULL,
    ref_value TEXT NOT NULL,
    ref_source TEXT,
    is_primary BOOLEAN NOT NULL DEFAULT false,
    metadata JSONB NOT NULL DEFAULT '{}'::jsonb,
    superseded_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    CONSTRAINT chk_wechat_article_external_refs_type
        CHECK (ref_type IN (
            'published_url',
            'canonical_url',
            'wechat_msg_id',
            'wechat_biz_mid_idx_sn',
            'youmind_ref',
            'publish_target_ref',
            'manual_repair',
            'external_reference'
        )),
    CONSTRAINT chk_wechat_article_external_refs_value_nonempty
        CHECK (length(trim(ref_value)) > 0)
);

CREATE UNIQUE INDEX IF NOT EXISTS uq_wechat_article_external_ref_active
ON hermes.wechat_article_external_refs(ref_type, ref_value)
WHERE superseded_at IS NULL;

CREATE UNIQUE INDEX IF NOT EXISTS uq_wechat_article_external_ref_article_active
ON hermes.wechat_article_external_refs(article_id, ref_type, ref_value)
WHERE superseded_at IS NULL;

CREATE INDEX IF NOT EXISTS idx_wechat_article_refs_article_created
ON hermes.wechat_article_external_refs(article_id, created_at DESC);

CREATE INDEX IF NOT EXISTS idx_wechat_article_refs_type_value_active
ON hermes.wechat_article_external_refs(ref_type, ref_value)
WHERE superseded_at IS NULL;

CREATE INDEX IF NOT EXISTS idx_wechat_article_refs_primary
ON hermes.wechat_article_external_refs(article_id, ref_type, is_primary)
WHERE superseded_at IS NULL;
```

## Query Contracts

### `list_wechat_articles`

Allowed filters:

- `account`
- `topic_id`
- `run_id`
- `status`
- `publish_target`
- `date_from`
- `date_to`
- `limit`
- `offset`

Rules:

- Default `limit` should be 50; maximum should be 200.
- At least one filter is required unless an explicit bounded `limit` is supplied.
- Response returns article summary fields and may include compact external refs.
- Response must not include artifact `content_text`.

### `get_wechat_article`

Rules:

- Returns one article by `article_id`.
- Includes all active and historical external refs.
- Returns artifact ids only; consumers call `get_workflow_artifact_content` for bodies.
- Missing article returns structured `not_found`.

### `update_wechat_article_external_refs`

Rules:

- Requires `article_id` and at least one ref or patch field.
- May append refs, mark existing refs superseded, and patch `published_url`, `canonical_url`, `external_reference`, `status`, `published_at`, or `metadata`.
- If active `(ref_type, ref_value)` belongs to another article, return structured `conflict`.

## Error Mapping

| DB / Validation Condition | MCP Error |
|---|---|
| missing `account`, `run_id`, or usable idempotency key | `missing_required_field` |
| invalid `topic_id` / `article_id` UUID | `invalid_uuid` |
| FK run missing | `not_found`, field `run_id` |
| FK artifact missing | `not_found`, field matching artifact id |
| duplicate active external ref on another article | `conflict`, field `external_ref` |
| status not allowed | `invalid_field`, field `status` |
| published status without URL/reference | `invalid_field`, field `status` or `published_url` |
| missing migration/table/column | `schema_drift` |
| unexpected asyncpg failure | `database_error` |

## Open Implementation Notes

- UUID values are generated in repository/tool code so the migration does not require `CREATE EXTENSION pgcrypto` privilege.
- The first version should not perform deep WeChat URL parsing. Store parsed `biz/mid/idx/sn` only when caller provides it as an external ref or metadata.
- Do not expose delete tools in MVP. Archival is a status update, not row removal.
